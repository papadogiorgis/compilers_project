## Make Targets
* `make all`: Compiles all files and creates executables **alpha_compiler** and **alpha_vm**.
* `make alpha_compiler`: Compiles only the Alpha Language Compiler.
* `make alpha_vm`: Compiles only the Alpha Language Virtual Machine.
* `make clean`: Removes all executables, binary files, and files generated by the compiler.

## How to Run the Alpha Compiler (Manually)
* To run the compiler with an Alpha Language source file as an argument, use the command `./alpha_compiler test.asc`. You can use the flag `-debug-flag` to print the syntax analysis and the scope table.
* The intermediate code (quads) will be generated and saved to a file named `all_quads.txt` in the same directory.

## How to Run the Alpha Virtual Machine (Manually)
* After creating the binary file with the Alpha Compiler, you can run it using the command `./alpha_vm out.abc`
* You can use the flag `-debug-flag` for debugging.

## How to Run Using the Alpha Script

An easy way to compile and run an Alpha program is using the provided `alpha` shell script. This script automatically handles the build process, compiles your source file, executes it in the Virtual Machine, and cleans up the intermediate binary files.

**Prerequisite:** Ensure the script is executable by running:
`chmod +x alpha`

**Usage:**
* **Standard Run:** `./alpha <source_file.asc>`
* **Debug Run:** `./alpha -debug-flag <source_file.asc>`

**What the script does automatically:**
1. Silently runs `make all` to ensure the compiler and VM are executable.
2. Runs `./alpha_compiler` on your target file (with *-debug-flag* if you are doing a debug run).
3. If successful, runs `./alpha_vm` on the generated `out.abc` file (with *-debug-flag* if you are doing a debug run).
4. Safely removes the intermediate `out.abc` file once execution finishes.
